4.3. Ontwikkeling

4.3.1. Keuze voor de opleiding

Waarom programmeren?

Ik heb voor de opleiding programmeren gekozen omdat de passie voor IT er eigenlijk altijd al is geweest. Al in het middelbaar twijfelde ik heel erg tussen mijn toenmalige studie en een richting in de IT. Destijds heb ik een andere keuze gemaakt, maar de interesse in software en codering is nooit weggegaan. Na mijn middelbare school ben ik eerst gaan werken, maar tijdens die werkervaring merkte ik dat ik mijn passie voor technologie miste. Ik vond het tijd om eindelijk iets te gaan doen wat ik echt leuk vind. Daarom heb ik de knoop doorgehakt om alsnog voor deze opleiding te kiezen. Ik wil nu de kans grijpen om alles te leren over het bouwen van software en van mijn hobby mijn beroep te maken. Het voelt als de juiste stap om nu vol voor het programmeren te gaan.

Waarom dit beroep?

Ik geloof dat een toekomst in de IT goed bij mij past, omdat ik door mijn werkervaring heb geleerd wat het betekent om hard te werken en verantwoordelijkheid te nemen. Doorzetten wanneer iets niet meteen lukt is voor mij vanzelfsprekend geworden, en die mindset neem ik mee in mijn ontwikkeling als programmeur. Programmeren vraagt geduld, focus en logisch denken, en juist die combinatie spreekt mij enorm aan. Ik merk dat ik zeer geconcentreerd kan blijven wanneer ik met code bezig ben, omdat het onderwerp mij oprecht interesseert. Waar ik vroeger twijfelde over mijn richting, weet ik nu zeker dat ik graag programmeer en haal ik veel voldoening uit het oplossen van complexe puzzels en het stap voor stap bouwen van een werkend programma. Daarnaast leer ik graag bij; in de IT-sector sta je nooit stil, en dat zie ik als een kans om mezelf continu te verbeteren. Met mijn motivatie, doorzettingsvermogen en leergierigheid ben ik dan ook vastberaden om mezelf te ontwikkelen tot een bekwame programmeur.

Overzicht van de OLR's en mijn huidige status per rol.

Rol: Ontwerper

Situering: Momenteel sta ik op een punt waarbij ik de basis van webdesign goed onder de knie heb. Wat mij al erg goed lukt, is het visueel vertalen van mijn ideeën naar code; ik kan een website er precies zo laten uitzien als ik in mijn hoofd heb door gericht gebruik te maken van CSS. Wat ik echter nog erg moeilijk vind, is de responsiviteit. Het zorgen dat een layout flexibel meebeweegt met verschillende schermformaten, zoals tablets en smartphones, is een grote uitdaging waar ik nog veel in moet bijleren.

4.3.4 Voorbeelden:
  • Layout-structuur bepalen: Bij het bouwen van dit portfolio heb ik eerst bepaald waar de navigatie, de inhoud en de footer moesten komen.
  • Styling via CSS: Ik heb een eigen stijl ontworpen voor mijn pagina's door kleuren, lettertypes en marges te definiëren.
  • Testen op mijn telefoon: Gekeken hoe mijn site er op mijn gsm uitziet om te zien wat er nog niet klopt.

Rol: Programmeur

Situering: Dit is de rol die ik het allerleukste vind, omdat ik hier echt met code bezig ben. Het lukt me al goed om kleine programma's te schrijven in C# en opdrachten in de Console op te lossen. Ik vind het fijn om stap voor stap een probleem aan te pakken tot de code werkt. Wat ik nog moeilijk vind, is om de code heel "netjes" te schrijven volgens alle regels van professionele programmeurs. Soms is mijn code nog wat te lang of onoverzichtelijk, en daar wil ik nog aan werken.

4.3.4 Voorbeelden:
  • C# oefeningen maken: Opdrachten gemaakt waarbij het programma berekeningen doet of tekst op het scherm zet.
  • Werken met GitHub: Geleerd hoe ik mijn code moet uploaden naar GitHub zodat mijn werk veilig bewaard blijft.
  • Code verbeteren: Soms schrijf ik een stuk code opnieuw omdat ik een kortere of betere manier heb gevonden.

Rol: Tester

Situering: Ik vind het leuk om te zoeken naar foutjes in mijn code, want dat voelt als een puzzel die ik moet oplossen. Het lukt me al goed om met de 'debugger' in Visual Studio te kijken waar mijn programma precies vastloopt. Wat ik nog moeilijk vind, is om vooraf een goed plan te maken voor wat ik precies allemaal moet testen. Nu begin ik vaak gewoon met klikken en kijken wat er gebeurt, in plaats van alles heel officieel aan te pakken.

4.3.4 Voorbeelden:

  • Feedback vragen: Opdrachten aan een medestudent laten zien om te kijken of zij begrijpen hoe het werkt.

Situering: Door mijn eerdere werkervaring vind ik het makkelijk om met mensen te praten en samen te werken. Wat me al heel goed lukt, is om aan medestudenten uit te leggen hoe ik een bepaald probleem heb opgelost. Ook durf ik om hulp te vragen als ik zelf vastloop. Wat ik nog lastig vind, is om de juiste Engelse vaktermen te gebruiken als ik over mijn code praat. Soms weet ik wel wat ik bedoel, maar vind ik het moeilijk om de officiële namen van de functies te onthouden.

4.3.4 Voorbeelden:

Levenslang lerende IT-professional

Situering: Ik ben iemand die heel erg gemotiveerd is om een goede programmeur te worden. Wat me al goed lukt, is om zelf informatie op te zoeken op het internet als ik vastloop met een opdracht. Ik ben streng voor mezelf en kijk altijd hoe ik mijn code de volgende keer beter kan maken. Wat ik nog moeilijk vind, is om te kiezen wat ik eerst moet leren. 4.5. Reflectie

4.5.1. AIM-reflectie

Intro: Ik ben Kobe, en mijn start aan deze opleiding was een heel bewuste keuze. Na een aantal jaren gewerkt te hebben in een andere sector, merkte ik dat ik toe was aan iets nieuws. Ik zocht een job waarbij ik echt mijn hoofd moet gebruiken en waar ik elke dag iets nieuws kan leren. Ik ben altijd al een puzzelaar geweest; iemand die niet opgeeft voor een probleem is opgelost. De wereld van IT is daarvoor de perfecte plek. Hoewel het spannend was om weer te gaan studeren, was ik vanaf de eerste dag enorm gemotiveerd om te laten zien wat ik in mijn mars heb.

Positief: Wat ik echt vet vond aan WPL1 is dat het niet alleen maar saaie theory is, maar dat je meteen zelf dingen mag bouwen. Vooral het Scratch-project vond ik heel leuk om te doen. Omdat ik van puzzelen hou, was het cool om zelf te bedenken hoe ik die code kon schrijven om het spel te laten werken. Ook het bouwen van mijn eigen CV-website met HTML en CSS was een hoogtepunt. Het geeft een kick om iets te maken dat er ook echt goed uitziet op een scherm.

Negatief: Het minst leuke waren de momenten dat ik vastliep. Vooral in de startfase maakte ik soms domme foutjes die ik pas veel later doorhad, wat best frustrerend was. Bij C# merkte ik hoe streng die code is: één kleine typefout en niks werkt meer. Mijn grootste uitdaging was mijn eigen perfectionisme. Ik wilde alles meteen 100% perfect hebben, waardoor ik soms veel te lang bleef hangen bij een klein detail en te weinig tijd overhad voor de rest.

Inzichten: Ik heb vooral geleerd dat je eerst moet nadenken en plannen voordat je als een gek begint te typen. Vroeger begon ik gewoon, maar nu weet ik dat je een opdracht beter eerst in kleine stukjes kunt hakken. Ook heb ik geleerd hoe handig 'debuggen' is; met breakpoints zie je precies waar je de mist in gaat. Mijn belangrijkste lesson? Je hoeft niet alles meteen perfect te doen. Het is veel beter om eerst een werkende basis te maken en het daarna pas mooier te maken.

Succes: Ik ben er het meest trots op dat ik alle opdrachten gewoon heb gefixt. Vooral bij Scratch ben ik blij dat ik meer heb gedaan dan wat moest, gewoon om te zien of het me lukte. Ook dit portfolio en mijn online CV vind ik een succes; ik heb nu eindelijk iets professioneels dat ik aan anderen kan laten zien. Het feit dat ik nu zelf code kan schrijven en begrijp wat ik doe, geeft me veel vertrouwen voor volgend semester. Ik zit hier echt op mijn plek.

Werkplekleren 2

De projectcase voor Werkplekleren 2, inclusief een uitgebreide reflectie op mijn eigen aandeel binnen het team.

Beschrijving van de case

Voor het vak Werkplekleren 2 kregen we de uitdaging om from scratch een volwaardig, data-gedreven product- en bestelbeheersysteem te ontwerpen. Wat deze case direct interessant maakte, was de technische gelaagdheid en de absolute noodzaak voor een professionele softwarearchitectuur. We kregen de opdracht om een robuuste backend op te zetten met een SQL Server database, een ASP.NET Core API en een afgescheiden Class Library voor de business logica, gecombineerd met een WPF App.

Het systeem moest twee strikt gescheiden gebruikersstromen faciliteren: bezoekers en medewerkers. Aan de ene kant moesten we een responsive platform bouwen waar bezoekers naadloos door producten kunnen filteren. Aan de andere kant was er behoefte aan een krachtig dashboard voor medewerkers met uitgebreide CRUD-operaties en datavisualisaties.

Tot slot lag er een enorme focus op security en identity management (Identity, rollen, triggers). Deze case was de perfecte brug tussen abstracte concepten en een tastbaar product.

Mijn aandeel hierin

Binnen dit project heb ik me vooral vastgebeten in de fundering van de applicatie en de datastromen, beginnend bij de database. Ik heb de tijd genomen om een goed gestructureerde SQL Server database te ontwerpen. Door vooraf kritisch na te denken over de relaties, hebben we een solide en schaalbare basis gelegd.

Vervolgens heb ik de brug gebouwd door de API-connectie (ASP.NET Core) van de backend naar de frontend te schrijven. Het doel was om de data veilig, snel en zonder haperingen van de database naar het scherm te krijgen met heldere endpoints.

Tot slot heb ik me gefocust op de vertaalslag naar de eindgebruiker: het management dashboard. Ik heb ervoor gezorgd dat dit extreem gebruiksvriendelijk is ingericht. Ruwe data is via externe libraries omgezet in strakke statistieken en grafieken, zodat medewerkers in één oogopslag de juiste beslissingen kunnen nemen.
